## 1. 目标说明

本计划面向具备 Java、PHP 和 SQL 背景的开发者，当前目标是在 1 个月内完成一轮高强度转向训练，快速补齐 Python 基础、工程化能力、数据处理能力与 AI / 大模型应用开发能力，并产出可用于求职展示的项目成果。

### 适用人群
- 已有后端开发经验
- 具备一定编程基础
- 希望转向 Python / AI / 大模型方向
- 当前看得懂基础 Python 代码，但缺少实际编写经验
- 每天可投入至少 4 小时

### 当前约束
- 学习周期只有 1 个月，必须优先高频技能
- 目标岗位聚焦 AI / 大模型 / RAG / Agent 相关岗位
- 计划在 1 个月后开始投递简历
- 目标招聘环境以新一线城市为主
- 可接受少量付费 API 或工具成本
- 学习资料尽量中文优先
- 需要可见进度、可打卡、可拆分的短任务
- 暂不投入前端重度开发

### 最终目标
- 能在 Python 中独立完成脚本、模块、服务和简单工程化组织
- 能完成一个可展示的 RAG / Agent 方向项目，并具备 README、截图、部署地址
- 能解释召回、切分、重排、评估等关键概念，并落到实际实现
- 能围绕项目输出简历素材、项目讲解材料和面试准备清单

## 2. 学习原则

1. 以就业为导向，不做无目标泛学。
2. 先掌握高频知识，再扩展低频知识。
3. 每个阶段都要有可验证输出。
4. 学习与实战并行，避免只看不写。
5. 每完成一个子任务，都要更新进度状态。
6. 优先复用既有后端经验，把 Java / PHP / SQL 迁移成 Python 工程能力。
7. 每个任务尽量控制在 30 到 90 分钟，降低中断成本。
8. 避免犯教学习惯错误：不要习惯按“最小可行路径”教学——先让测试跑起来，再说进阶。一定保证本 SPEC 的完整性要求
9. 教学内容，除了代码步骤，还要配合一些步骤讲解，不要跳过了概念讲解，直接给代码了。
10. 讲解前搭配上求职话术，面试怎么说。
11. Python版本是Python3.12+，不在需要诸如from typing import Optional的引用方式

## 4. 执行边界与阶段策略

### 阶段策略
- 阶段 A 只快速补基础，不做长时间语法泛学。
- 阶段 B 采用求职强化标准，覆盖 venv、pip、pytest、ruff、black、mypy、pre-commit、CI、Docker 基础。
- 阶段 C 聚焦文件 / 表格 / PDF 数据处理，接口能力只保留项目所需最小集合。
- 阶段 D 不停留在“会调用 API”，需要理解并实现 chunking、recall、rerank、evaluation。
- 阶段 E 采用“先小后大”的项目策略，所有阶段产出尽量复用到最终项目中。

### 不纳入本轮重点的内容
- 前端重度开发
- 深度学习训练与复杂数学推导
- 与求职目标弱相关的低频 Python 语法细节

### 推荐主项目方向
- 主项目：面向文档的 RAG 问答系统
- 子项目 1：文档解析与知识库构建工具
- 子项目 2：多模型 / 提示词 / 日志封装模块
